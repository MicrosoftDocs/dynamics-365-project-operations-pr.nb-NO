<span data-ttu-id="18ef9-105">En prosjektbasert kontraktlinje har *inkluderte* komponenter og *belastbare* komponenter.</span><span class="sxs-lookup"><span data-stu-id="18ef9-105">A project-based contract line has *included* components and *chargeable* components.</span></span>

<span data-ttu-id="18ef9-110">Et delsett av de inkluderte komponentene kan merkes som belastbarte ved hjelp av feltet **Faktureringstype**.</span><span class="sxs-lookup"><span data-stu-id="18ef9-110">A subset of the included components can be marked as chargeable using the **Billing Type** field.</span></span> <span data-ttu-id="18ef9-111">Feltet **Faktureringstype** er et alternativsett som tillater følgende verdier i konteksten for en kontraktlinje:</span><span class="sxs-lookup"><span data-stu-id="18ef9-111">The **Billing Type** field is an option-set that allows the following values in the context of a contract line:</span></span>

  - <span data-ttu-id="18ef9-113">Ikke-belastbar</span><span class="sxs-lookup"><span data-stu-id="18ef9-113">Non-chargeable</span></span>

<span data-ttu-id="18ef9-114">Belastbare komponenter kan defineres for oppgaver, roller og transaksjonskategorier.</span><span class="sxs-lookup"><span data-stu-id="18ef9-114">Chargeable components can be defined on tasks, roles, and transaction categories.</span></span>
